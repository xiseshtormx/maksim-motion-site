# KAIROS — Maksim Motion Site

Сайт-портфолио Максима Рыжикова: motion-дизайн, видеомонтаж, sound и визуальная айдентика.

Проект работает как современная рекламная витрина специалиста: показывает авторский стиль, направления работы и портфолио, формирует доверие и приводит посетителя к проектному брифу.

## О проекте

Сайт создаётся для Максима Рыжикова — специалиста в сфере motion-дизайна. В отличие от обычной статичной страницы, проект сочетает выразительную визуальную подачу, адаптивные анимации, реальные видеоработы, динамический расчёт заявки и управляемое портфолио.

Для разработчика это практический frontend-кейс с реальным заказом: структура landing page, дизайн-система, адаптивная вёрстка, оптимизация медиа, CMS и подготовка production-развёртывания.

## Цели проекта

- показать личный бренд и визуальный почерк motion-дизайнера;
- представить Reels, motion-айдентику и полноформатные видео;
- сделать понятный путь от просмотра работ до заявки;
- сохранить высокое качество на десктопе и мобильных устройствах;
- позволить владельцу самостоятельно обновлять портфолио;
- подготовить сайт к автоматической публикации через GitHub и Netlify.

## Локальный запуск

```sh
npm install
npm run dev
npm run build
npm run preview
```

Production-сборка появляется в `dist/`. Требуется Node.js 22 или новее.

## Где лежит контент

- `src/data/site.json` — SEO, имя, email и основной текст сайта;
- `src/data/projects/*.json` — три карточки портфолио блока 04;
- `public/uploads/` — загруженные обложки и видео;
- `src/assets/maxim-ryzhikov.webp` — фотография Максима;
- `public/admin/config.yml` — доступные в CMS поля;
- `OWNER-GUIDE.md` — инструкция для владельца.

Карточка проекта поддерживает изображение, видео с постером, галерею материалов и подборку вертикальных Reels.

## Панель управления

Интерфейс подготовлен по адресу `/admin/` на Decap CMS. Владелец входит через GitHub и редактирует только три карточки блока «04 / SELECTED WORKS»: изображения айдентики, подборку Reels и полноформатные видео. Остальные секции сайта в панели скрыты.

После публикации CMS обновляет JSON и медиа в GitHub. Netlify замечает commit, запускает `npm run build` и автоматически обновляет сайт.

## Активация CMS на Netlify

1. Импортировать репозиторий `xiseshtormx/maksim-motion-site` в Netlify.
2. Выполнить первый deploy: параметры уже находятся в `netlify.toml`.
3. Создать GitHub OAuth App с callback `https://api.netlify.com/auth/done`.
4. В Netlify открыть `Project configuration → Access & security → OAuth` и установить GitHub provider с Client ID и Client Secret.
5. Добавить GitHub-аккаунт Максима collaborator-ом репозитория с правом записи.
6. Открыть `https://ваш-домен/admin/`, войти через GitHub и обновить тестовую карточку.

Отдельный пароль внутри сайта не хранится: доступ контролируют GitHub и OAuth.

## Заявки

Форма `project-request` передаёт контакты, комментарий, формат, объём, предварительный бюджет, желаемую и рекомендованную даты. Проверенные заявки хранятся в `Netlify → Forms`, а функция `netlify/functions/project-lead-email.mjs` отправляет Максиму фирменное HTML-письмо через Resend.

Для активации красивых писем:

1. Зарегистрировать аккаунт на Resend и создать API key.
2. В Netlify открыть `Project configuration → Environment variables`.
3. Добавить `RESEND_API_KEY`, `KAIROS_LEADS_TO` и `KAIROS_MAIL_FROM` по примеру из `.env.example`.
4. Запустить новый deploy и отправить тестовую заявку с сайта.
5. Если в Netlify уже включено стандартное email-уведомление Forms, отключить его после успешного теста, чтобы письма не дублировались.

До подключения собственного домена Resend позволяет отправлять с `onboarding@resend.dev` только на адрес владельца аккаунта. После проверки домена укажите, например, `KAIROS <brief@kairos-motion.ru>`.

Для будущего Telegram-уведомления можно добавить Netlify Function. Токен бота и `chat_id` должны храниться только в переменных окружения хостинга.

## Перед публикацией на основном домене

- заменить тестовый `hello@kairos-motion.ru` на реальный адрес;
- добавить утверждённую OG-обложку;
- заполнить описания материалов (`alt`);
- использовать WebP/JPEG для постеров и MP4 H.264 для видео;
- держать короткие превью по возможности до 8–10 МБ;
- проверить тестовую заявку и публикацию через `/admin/`.

Подробная инструкция для Максима находится в `OWNER-GUIDE.md`.
