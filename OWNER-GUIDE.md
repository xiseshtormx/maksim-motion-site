# Как Максиму обновлять сайт

Панель управления находится отдельно от публичного лендинга: `https://ваш-домен/admin/`. Она не видна посетителям. Для безопасной публикации нужен бесплатный GitHub-аккаунт, добавленный соавтором репозитория; работать с кодом и командами Git не потребуется.

В панели доступны только три карточки блока «04 / SELECTED WORKS». Остальные секции сайта зафиксированы и случайно изменить их через админку нельзя.

## Обновить работу

1. Откройте `/admin/` на опубликованном сайте и нажмите вход через GitHub.
2. Перейдите в раздел «Портфолио — блок 04» и выберите нужную карточку.
3. Проверьте название, формат и короткое описание.
4. Загрузите обложку или видео. Для видео желательно добавить отдельный постер.
5. Для вертикального Reels выберите «Показать полностью», для горизонтального видео — «На весь кадр».
6. Заполните клиента, год и «Описание изображения» — это помогает посетителям, поисковикам и экранным дикторам.
7. При необходимости вставьте ссылку на полный ролик на YouTube, Vimeo, Rutube или Behance.
8. Включите «Показывать на сайте» и нажмите Publish.

Обновлённая работа появится после автоматической пересборки сайта. Обычно это занимает несколько минут.

## Что происходит после Publish

1. Decap CMS обновляет JSON-файл карточки и загружает медиа в `public/uploads/`.
2. Decap CMS через GitHub OAuth создаёт обычный commit от имени авторизованного редактора.
3. Netlify замечает новый commit и запускает `npm run build`.
4. После успешной сборки обновлённая галерея автоматически появляется на основном домене.

Если в данных проекта будет ошибка, схема контента остановит сборку, а предыдущая рабочая версия сайта останется опубликованной.

## Скрыть проект

Откройте запись проекта, выключите «Показывать на сайте» и опубликуйте изменение. Файл останется в системе, но карточка исчезнет с сайта.

## Получать заявки

Кнопки «Обсудить проект», «Обсудить задачу» и «Начать проект» открывают одну форму. После первого deploy проверьте на хостинге, что форма `project-request` появилась в списке форм, и включите уведомления на рабочую почту Максима.

## После подключения домена

- проверить canonical и OG-обложку через исходный код главной страницы;
- открыть `https://ваш-домен/sitemap-index.xml`;
- добавить домен в Google Search Console и Яндекс Вебмастер;
- отправить sitemap и запросить переобход главной страницы;
- проверить реальную заявку с телефона и получение письма Максимом.

## Подготовка медиа

- Обложка: WebP или JPEG, ширина 1600–2000 px, ориентир до 400 КБ.
- Видео: MP4 H.264 без звука для превью, 1080p, ориентир до 8–10 МБ.
- Постер: кадр из видео в WebP/JPEG, чтобы карточка не была пустой во время загрузки.
- Полный ролик со звуком лучше размещать на Vimeo, YouTube или Behance и добавлять ссылкой в поле «Ссылка на полный кейс».

Не загружайте исходники монтажа, личные документы и материалы клиента, на публикацию которых нет разрешения.
